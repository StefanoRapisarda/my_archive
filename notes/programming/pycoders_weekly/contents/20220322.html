Abount PEP

DEF: PEPs are documents that propose new Python features and serve as official documentation for the feature while the Python community debates its acceptance or rejection. PEP stands for Python Enhancement Proposal;

Among all PEPs, PEP8 is Python's official style guide. Some of the "rules" in PEP8 are:

  • Use four spaces instead of a tab character for indentation;
  • The maximum line length is 79 characters;
  • Functions and variable names should follow the lowercase_with_underscores style;
  • Class names should use the CapitalizeWords style;
  • Write constants in the UPPER_CASE_WITH_UNDERSCORE style;

There are tools to check if you are "respecting" the rules:

  • flake8: install it with pip install flake8 and run it with flake8 myscript.py;
  • You can set VS Code to lint your code with flake8;
  • black is an auto-formatter that you can install with pip install black. black --check myscript.py will check your script to see if there is any change to do. black --diff myscript.py is to check which changes black would make. black myscript.py will automatically format the file.

Making your code more "Pythonic"

Apart from following PEP8 for your formatting, there are few things to remember to make your python code more efficient:

  • Instead of iterating over an index, try to iterate over items in a list (or anything iterable) directly;
  • Use list comprehension when possible (I never noticed that list comprehension syntax is very symilar to the set-builder notation) for writing sets ("insiemi") in mathematics);
  • Use built-in functions when possible. A good example is finding the amount of words starting with A in a list: sum([item.startswith("A") for item in items]);
  • Remember that in python True and False are 1 and 0 in disguise;
  • Remember you can also write dictionary comprehensions list list comprehensions;
  • Use sets to remove duplicates from a list: unique_items = list(set(items)) .
    Sets can also be used to get differences between two... sets of items;
