React

Definitions

React is another solution for SPAs, it was released by Facebook later in 2013. As Angular (Google), Ember, Knockout, and Backbone, React is used to create entire web applications in JavaScript [2].

React project Set-Up

Step1: creating your React environment

npx create-react-app your_react_project

or

npm init react-app your_react_project

npx stands for node package executor. You can find more information about create-react-app here. The previous command will create a folder with the name of your app. Inside the folder there are a series of sub-folders and files containing all you need to start your React project. In particular:

  • A README.md markdown file. This is supposed to briefly describe your project (GitHub style);
  • package.json, a file containing a list of node package dependencies and oteher project configurations. Going through your package.json file, you will also find the list of scripts you can use under the scripts property;
  • public, a directory holding all our development files, including the public/index.html file (the single page of our SPA) and the sub-directory src/ containing all the JavaScript code. In particular the src/App.js file is used to implement React components. Of course, this is the setup configuration, as you are going to develop your project you may want to organize your components into different files;
create-react-app will also configure tools such us webpack and Babel, so that you do not need to worry about those.

Step2: Start the app in developer mode

npm start

Step3: Run the tests

npm test

Step4: Build the aoo for production

npm run build

This command will create a build directory that can be used to deploy your application;

About JSX

JSX is a syntax extension to JavaScript. It is not mandatory to use JSX in order to use React (as it is not mandatory to use React to implement SPA), but it makes life a lot easier.

JSX main features:

  • JavaScript variables (variables that have been declared at a certain point either using const or let) can be used inside JSX wrapping them around curly braces (e.g. {my_variable}). You can actually put entire Javascript expressions inside curly braces;
  • JSX expressions can be used inside conditaional statements, for loops, assigned to variables, accepted as arguments, and returned by functions. This is all because JSX statements are translatedinto JavaScript during compilation;
  • In JSX object properties use the camelCase convention, the same of JavaScript functions. So that HTML class and tabindex become className and tabIndex in JSX;

Practical tips

From my very first coding of mdn example in codepen

  • Remember to declare all the variable either with let or const;
  • It's ReactDOM not ReactDom;
  • Do you want to place a React component inside an html element? Then:
    • Give that element a unique ID (myid);
    • Create a React root for the supplied container using root = ReactDOM.createRoot(document.getElementById('myid'));;
    • Use the just created root to render the react element using root.render(< MyReactElement args={whatever} />)
  • !!! When you create a list of elements, you should give to each element a unique ID, so that React can identify which item changed.
